PPI-RF-DL

This repository contains code for protein–protein interaction (PPI) prediction
using sequence-based features with deep learning and random forest models.


Data description

The PPI data used in this project were obtained from the STRING database.

Database: STRING  
Version: v12.0  
Organism: Homo sapiens  
NCBI taxonomy ID: 9606  
Original file: 9606.protein.links.detailed.v12.0.txt  
Filtering rule: combined_score >= 400  

Due to GitHub file size limitations, the original STRING interaction file is not
included in this repository.


Data download

The raw STRING interaction data can be downloaded from:

https://string-db.org/cgi/download

Please select Homo sapiens (9606) and the protein.links.detailed dataset (v12.0).
After downloading, apply the same filtering rule used in this project.


Data preprocessing

The data were processed as follows:

1. Interactions with combined_score >= 400 were retained.
2. Protein identifiers were normalized by removing version suffixes.
3. Positive PPI pairs were extracted from the filtered data.
4. Negative samples were generated by random pairing of proteins.
5. Protein sequences were loaded from FASTA files.
6. Sequence features were generated using amino acid frequency embeddings.

All preprocessing steps are implemented in 1.py.


Feature representation

Each protein is represented by a 20-dimensional amino acid frequency vector.
For each protein pair, the two vectors are concatenated to form a 40-dimensional
feature vector.


Models

Two models are used in this project.

1. A deep learning model implemented with PyTorch.
2. A random forest model implemented with scikit-learn.


Evaluation

Model performance is evaluated using accuracy, precision, recall, F1-score, and
ROC AUC. Evaluation results are saved in the results directory.

Reproducibility

All experiments can be reproduced by downloading the STRING dataset (v12.0),
applying the same filtering threshold, and running the provided scripts.
